# PEEC Content Opportunity Brief

## Purpose

Use this skill when PEEC data shows an opportunity that content can address:
weak visibility, low share of voice, poor sentiment, weak answer position,
missing citations, or competitor ownership of important prompts.

## Inputs

- Brand report metrics for the target brand and competitors.
- Prompt, tag, citation, and cited URL data from MCP when available.
- Existing pages, landing pages, docs, articles, product pages, comparison
  pages, case studies, or external profiles.
- Relevant findings from `$seo-geo-strategy` when a broader strategy exists.

## Brief Structure

Create a brief with:

- **Target metric:** visibility, share of voice, sentiment, position, citations,
  or prompt inclusion.
- **Target prompt/topic:** the exact prompt cluster, category, audience, or
  query theme.
- **Current gap:** what PEEC says is weak and who is winning instead.
- **Page decision:** refresh existing page, consolidate pages, create a new
  page, add evidence, update external profile, or pursue third-party citation.
- **Content requirements:** claims, facts, product details, methodology,
  original data, comparison table, FAQ, entity facts, schema, author/reviewer,
  and citations.
- **Extraction requirements:** concise answer blocks, descriptive headings,
  stable URL, tables where useful, alt text/captions, and consistent entity
  naming.
- **Measurement:** which PEEC numbers should be checked next week and what
  leading indicator counts as progress.

## Rules

- Prefer improving an already relevant page over creating a shallow new page.
- Do not propose mass-generated pages.
- Require evidence or first-hand detail for claims intended to improve AI answer
  citations.
- If sentiment is the target, specify which language or entity facts need to
  change, not just "make it more positive."
